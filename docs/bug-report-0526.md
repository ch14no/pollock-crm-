# バグ報告 対応状況レポート（2026-05-26 受領分）

## バグ一覧

### ① 活動記録を1件登録したら、同じ内容が2件反映される（アーティストキャリア）

**ステータス: ✅ 修正完了**

> **原因**: 活動を作成した直後にローカルストアとDBの両方に同じ活動が存在し、画面上で重複表示されていた。
>
> **対応**: 顧客詳細ページのタイムライン表示で、DBとローカルの活動を結合する際に重複除去処理を追加。同じIDの活動はDB側を優先して1件のみ表示するよう修正。

---

### ② タスク管理のカンバン移動が全員に反映されない

**ステータス: ✅ 修正完了**

> **原因**: タスクのステージ移動がローカルのブラウザストレージにのみ保存されており、他のユーザーのブラウザには反映されていなかった。
>
> **対応**:
> - ドラッグ&ドロップ時に Supabase の `task_meta` テーブルへステージIDを保存するよう修正
> - タスク管理ページ起動時にDBからステージ情報を読み込み、全ユーザーの表示を統一
> - **別途 SQL 実行が必要でした**（実施済み）:
>   ```sql
>   ALTER TABLE task_meta ADD COLUMN IF NOT EXISTS kanban_stage_id TEXT;
>   ```

---

### ③ 商談カンバン — 顧客ページから遷移直後はなにも表示されないが、追加を押すと出てくる

**ステータス: ✅ 修正完了**

> **原因**: `KanbanBoard` コンポーネントはマウント時の初期データのみで描画していたため、非同期で取得した商談データが後から届いても画面に反映されなかった。
>
> **対応**:
> - DB からのデータ取得完了後に `KanbanBoard` を再マウントして最新データを表示するよう修正
> - ドラッグ操作中は再マウントをスキップし、ドラッグ操作の邪魔をしないよう制御

---

### ④ すでに顧客登録されている方のタグが少ない

**ステータス: ℹ️ 仕様説明・対応方法あり**

> **原因**: 表示制限のバグではありません。登録時にタグが設定されていなかったか、インポート時にタグ列のマッピングが「取り込まない」になっていた可能性があります。
>
> **対応方法**:
> 1. **CSVで一括更新**: タグ列（`|` 区切りで複数指定）を含むCSVを用意し、インポート画面で「重複時: 更新する」を選択して実行すると、既存顧客のタグを上書き補完できます
> 2. **手動追加**: 顧客詳細ページ → 編集（✏️）→ タグ欄から追加
> 3. **住所から拠点タグを自動付与**: 住所データがある場合は、顧客一覧の「住所から拠点を設定」ボタンで都道府県に基づいた拠点タグを一括付与できます

### ⑤ 商談カンバン — 受注が50件になったときのレイアウト

**ステータス: ✅ 修正完了**

> **原因**: 受注・失注カラムの高さ制限がなく、件数が増えると画面外に縦長に伸びてしまっていた。
>
> **対応**: 各ステージカラムに最大高さを設定し、超えた場合はカラム内でスクロールできるよう修正。受注件数が何件になっても画面レイアウトが崩れません。

### ⑥ ダッシュボードのチーム受注額が人によって違う / 齋藤香奈さんのダッシュボードがバグっている

**ステータス: ℹ️ 仕様説明 ＋ 個別バグは要確認**

> **チームの受注額が人によって違う**:
> これは意図した動作です。サイドバーの「事業部」セレクタで選択している事業部が異なると、表示される受注額も事業部ごとに変わります。同じ事業部を選択した状態で比較すると同じ金額が表示されます。
>
> **齋藤香奈さんのダッシュボードがバグっている**:
> 具体的な症状（スクリーンショット、エラーメッセージ等）を共有いただければ調査いたします。考えられる原因としては:
> - 選択している事業部が他のメンバーと異なる
> - アカウントの所属事業部が正しく設定されていない（設定 → アカウント管理で確認可）

### ⑦ 商談カンバン — 削除機能がない・失注したものを復活させたい

**ステータス: ✅ 修正完了**

> **対応内容**:
>
> **削除機能を追加**
> - 商談をクリックして編集モーダルを開く → 下部に「削除する」ボタンを追加
> - 確認ダイアログが表示され、OKで商談を完全削除
>
> **失注からの復活**
> - 失注した商談をクリックして編集モーダルを開く → 「復活させる」ボタンを追加
> - 押すと最初のアクティブステージ（例: リード）に戻す
> - 失注後に再度成約するケースでも対応可能に
>
> **失注した商談の場所**
> - 商談カンバンの右端に「失注」カラムとして表示されています（スクロールで確認）

### ⑧ 2回連続で検索ができない

**ステータス: ✅ 修正完了**

> **原因**: ヘッダーの検索バーで1回検索すると入力欄がクリアされ、2回目の検索を入力して Enter を押しても同じURLへの遷移として扱われ更新されないケースがあった。
>
> **対応**: 検索後も入力欄に検索ワードを残すよう変更。ワードを修正してそのまま再検索できるようになった。
