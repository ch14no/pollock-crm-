## 2. 他事業部への影響調査

今回の改修はM&A事業部の要望が起点ですが、CRMは全社共通のため**全事業部に以下の変化があります**。コードの全画面・全クエリを対象に影響箇所を調査した結果です。

### 2-1. 【最重要】閲覧範囲の制限（全事業部・本日から有効）

011マイグレーション適用により、**「自分の所属事業部のデータのみ閲覧できる」**にルールが変わりました（従来は全社員が全事業部を閲覧可能）。

| 変わること | 影響を受ける人 | 詳細 |
|---|---|---|
| 事業部セレクタが所属事業部のみに | 全ユーザー（super_admin除く） | 他事業部のダッシュボード・顧客一覧・カンバンを見て回ることができなくなった |
| 他事業部の顧客・商談・活動がDBレベルで遮断 | 全ユーザー（super_admin除く) | UIだけでなくデータベース側で強制（セキュリティとして正しい状態） |
| **会社詳細ページの担当者一覧が自事業部のみに** | 全事業部 | 従来は1つの会社に紐づく全事業部の担当者が見えた（トスアップ先を探す手がかりだった）。今後は自事業部の担当者のみ表示 |
| **managerの活動閲覧が「同じ事業部のメンバー」のみに** | 全事業部のmanager | 従来managerは全社の活動履歴を閲覧できた。ダッシュボードのチーム集計・活動履歴が自事業部分のみになる |
| トスアップは引き続き機能 | — | 受け取った顧客の詳細は従来どおり閲覧可能。ただし**その顧客の商談・活動履歴は見えない**（露出最小化の方針。トスアップのメッセージ欄で文脈を伝える運用に） |

**変わらないもの**: 会社マスタ（全社共有のまま）／事業部一覧（トスアップの宛先選択に必要）／super_adminの全件閲覧／トスアップの送受信。

**懸念と対処**: 「あの顧客が見えなくなった」「会社詳細に他事業部の担当者が出ない」という問い合わせが出る可能性があります。これは不具合ではなく意図した変更です（下記の周知文案を参照）。業務上どうしても他事業部データの参照が必要なケースが出た場合は、(a) 該当ユーザーを複数事業部に所属させる、(b) トスアップで共有する、のいずれかで対応できます。

### 2-2. 【要対応】商品マスタの再登録（該当事業部のみ）

商品マスタ（商談登録時の「提案商品」選択肢）は、これまで**各自のブラウザにしか保存されておらず、ユーザー間で共有されていませんでした**（構造的な不具合）。010適用によりDB共有化された一方、**既存のブラウザ保存分は自動移行されません**。

- **対応**: 商品選択を使っていた事業部は、管理者が 設定＞商品マスタ管理 で (1) 商品を再登録、(2) 「商談画面に商品選択を表示する」をONに戻す（この設定もOFFに初期化されています）
- 商談ごとの選択商品も今後はDBに保存され、誰が見ても同じ内容になります

### 2-3. 【全事業部にプラス】不具合修正・改善

| 変更 | 影響 |
|---|---|
| 顧客編集の信頼性向上（②） | 全事業部。編集がユーザー間で確実に共有され、保存失敗時はエラーが表示されるように。**過去に「編集したはずが戻っている」と感じた事象はこれが原因の可能性大** |
| カンバンの読み込み安定化（⑥） | 全事業部。事業部切替時に他事業部のカードが紛れ込む事象を解消 |
| ステージ・タブ・カスタム項目定義の一元化 | 全事業部。従来は「設定画面を開いた端末」にしか最新定義が同期されない構造だったため、環境によっては**カンバンの列構成が今回から変わって見える場合があります**（DBに登録された正しい定義に統一されたため）。もし「以前あったはずの列に商談が見当たらない・先頭列に商談が寄っている」事象があれば管理者へ連絡（過去の暫定表示で保存された商談のステージ値をSQLで正規化して解消できます） |
| 名刺登録のスマホ対応（㉑） | 全事業部。外出先での顧客登録がしやすく |
| 一括タグ付けの改善 | 全事業部。住所→拠点タグの一括処理が、権限のない顧客1件で全体中断せず、失敗件数を表示して継続するように |

### 2-4. 全社周知の文案（そのまま使えます）

> 【CRMアップデートのお知らせ】
> 本日の更新で以下が変わりました。
> 1. **閲覧範囲**: CRMは自分の所属事業部のデータのみ表示されるようになりました。他事業部の顧客・商談は見えません（トスアップで受け取った顧客は引き続き閲覧できます）。他事業部との連携が必要な場合はトスアップ機能をご利用ください。
> 2. **商品マスタ**: 商談の「提案商品」を使っている事業部は、設定＞商品マスタ管理で商品の再登録と表示ONの再設定をお願いします（保存方式の改善に伴う一度きりの作業です）。
> 3. **不具合修正**: 顧客情報の編集が保存されない・反映されない不具合を修正しました。今後、保存に失敗した場合は画面にエラーが表示されます。
> 4. **スマホ対応**: 名刺からの顧客登録がスマホで使いやすくなりました（カメラ直接起動）。
