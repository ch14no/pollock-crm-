## 1. システム管理者の権限

システム管理者は以下の操作が可能です。

| 機能 | 一般ユーザー | マネージャー | システム管理者 |
|---|:---:|:---:|:---:|
| 顧客・商談の閲覧 | 自事業部のみ | 自事業部のみ | **全事業部** |
| 顧客・商談の作成・編集 | 自事業部のみ | 自事業部のみ | **全事業部** |
| トスアップの送受信 | ○ | ○ | ○ |
| ユーザーの追加・削除 | × | × | **○** |
| 事業部の設定変更 | × | × | **○** |
| カスタムフィールドの設定 | × | × | **○** |
| パイプラインステージの変更 | × | × | **○** |

## 2. 初期セットアップ

### 2-1. Supabase 環境変数の設定

本番環境（Vercel）では以下の環境変数を設定する必要があります。

| 変数名 | 説明 | 取得場所 |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | プロジェクトURL | Supabase → Settings → API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 公開キー | Supabase → Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | サービスロールキー（サーバーのみ） | Supabase → Settings → API |

> ⚠️ `SUPABASE_SERVICE_ROLE_KEY` は絶対に `NEXT_PUBLIC_` を付けないでください。フロントエンドに漏洩します。

### 2-2. DBマイグレーション（初回のみ）

Supabase の **SQL Editor** で以下を実行してください。

```sql
-- 顧客テーブルへの列追加
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS address TEXT;
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS department TEXT;

-- service_role に全テーブルへの権限付与（ユーザー管理機能に必要）
GRANT ALL ON ALL TABLES IN SCHEMA public TO service_role;
```

## 3. ユーザー管理

**設定 → アカウント管理** から操作します。

### 3-1. ユーザーの追加

1. 「ユーザーを追加」ボタンをクリック
2. 以下を入力して「追加」を押す

| 項目 | 説明 |
|---|---|
| 氏名 | アプリ内の表示名 |
| メールアドレス | ログインID（後から変更不可） |
| パスワード | 6文字以上。後から管理者がリセット可能 |
| 権限 | 下表参照 |
| 所属事業部 | チェックした事業部の顧客を編集できる |

**権限の種類**

| 権限 | 説明 |
|---|---|
| ユーザー | 所属事業部のみ閲覧・編集 |
| マネージャー | 所属事業部のみ閲覧・編集（表示上の区別のみ） |
| システム管理者 | 全事業部アクセス＋設定変更権限 |

> ⚠️ **所属事業部を設定しないと顧客の新規登録ができません。** ユーザー追加時に必ず設定してください。

### 3-2. ユーザーの編集

ユーザー一覧の各行で操作できます。

| ボタン | 機能 |
|---|---|
| ✏️ 鉛筆アイコン | 氏名・権限・所属事業部を変更 |
| 🔑 鍵アイコン | パスワードをリセット（新しいパスワードを本人に伝えること） |
| 🗑️ ゴミ箱アイコン | アカウントを削除（元に戻せません） |

> 自分自身のアカウントは削除できません。

## 4. 事業部管理

**設定 → 事業部設定** から操作します。

- 事業部名とカラーコードを設定できます
- サイドバーの事業部セレクタに反映されます

### パイプラインステージの設定

各事業部ごとに商談のステージを定義できます。

1. **設定 → パイプラインステージ** を開く
2. ステージ名を入力して追加
3. ドラッグで順番を変更
4. **受注・失注フラグ** を設定すると分析画面の集計に反映される

---

## 5. カスタムフィールド設定

事業部ごとに顧客の独自項目を追加できます。**顧客一覧のフィルターにも自動で反映されます。**

**設定 → カスタムフィールド** から操作します。

### フィールドタイプ

| タイプ | 説明 | フィルター対応 |
|---|---|:---:|
| テキスト | 自由記入 | × |
| 選択肢（select） | 選択肢から選ぶ | **○（チップ表示）** |
| 数値 | 数字のみ | × |

> 顧客一覧のフィルターに出したい項目は **選択肢型** で作成してください。

### 設定例

| フィールド名 | タイプ | 選択肢の例 |
|---|---|---|
| 案件区分 | 選択肢 | 人材 / 案件 / 両方 |
| クライアント区分 | 選択肢 | ベンダー / エンドユーザー / ソフトハウス / 受託会社 |
| 技術区分 | 選択肢 | インフラ / 開発 / 両方 |
| 単価目安（万円） | 数値 | — |
| 備考 | テキスト | — |

## 6. CSVインポート・エクスポート

**インポート・エクスポート** メニューから操作します。

### 6-1. インポート手順

1. **CSVファイルをアップロード**
   - 文字コード: UTF-8 推奨
   - Excelで作成した場合は「名前を付けて保存 → CSV UTF-8」で保存すること
2. **マッピング設定**: CSVの列と顧客フィールドを対応付ける
   - 設定は事業部ごとに自動保存され、次回から自動適用される
   - 対応しない列は「取り込まない」を選択
3. **プレビューで確認**: 先頭5件のデータを確認
4. **重複時の処理を選択**（下記参照）
5. **実行**

> ⚠️ インポート中はページを離れないでください（途中で止まります）

### 6-2. 重複時の処理

名前 ＋ メールアドレスが両方一致した場合を「重複」と判定します。

| 設定 | 動作 |
|---|---|
| **スキップ**（デフォルト） | 既存顧客を変更しない |
| **更新する** | 住所・電話番号など、CSVに値がある項目だけ上書き補完。空欄の列は既存データを消さない |

住所を後から補完するCSVを入れる場合は「更新する」を選択してください。

> メールのみ同じで名前が違う場合は別人として登録されます（共通メール対応）。

### 6-3. 対応フィールド一覧

| CSV列名の例 | 対応フィールド |
|---|---|
| 氏名、担当者名、name | 担当者名（**必須**） |
| 会社名、企業名 | 会社名 |
| メール、mail | メールアドレス |
| 電話、TEL、phone | 電話番号 |
| 役職、position | 役職 |
| 部署名、department | 部署名 |
| 住所、address | 住所 |
| メモ、備考、note | メモ・備考 |
| タグ | タグ（`|` 区切りで複数指定） |

### 6-4. 住所から拠点タグを自動設定

CSVインポートで住所データを取り込んだ後、顧客一覧で一括タグ付けできます。

1. 顧客一覧の **「住所から拠点を設定」** ボタンをクリック
2. 各拠点に対応するキーワードを設定（カンマ区切り）
   - 例: 東京 → `東京都,神奈川県,埼玉県,千葉県`
3. 「実行」で住所に一致する顧客に自動でタグを付与

### 6-5. エクスポート

| 操作 | 方法 |
|---|---|
| 全件エクスポート | ヘッダー右上の「CSVエクスポート」 |
| 選択件数のみ | チェックボックスで選択 → 下部バーの「CSVエクスポート」 |

## 7. 顧客管理の運用

### 7-1. 顧客の登録方法

| 方法 | 説明 |
|---|---|
| 手動登録 | 「新規顧客」ボタン → フォーム入力 |
| 名刺スキャン | 「名刺から登録」→ 画像アップロード → OCR結果を確認・修正 |
| CSVインポート | 6章参照 |

### 7-2. 顧客ステータス

顧客詳細ページ上部のアイコンでトグル設定します。顧客一覧のフィルターで絞り込みに使えます。

| アイコン | 名称 | 用途例 |
|---|---|---|
| ★ | 重点顧客 | 優先フォロー対象 |
| ♥ | 仲良くしたい | 関係強化中 |
| ↑ | 今後伸びる | 将来性が高い |
| 💀 | NG/BL | 対応不可・ブラックリスト |
| 👍 | 自慢/引き抜き | 優秀な人材・紹介したい顧客 |

### 7-3. 顧客一覧の表示モード

| モード | 説明 |
|---|---|
| リスト | 1行1顧客の標準表示 |
| カード | カードタイル表示 |
| 会社別 | 会社でまとめてアコーディオン表示。クリックで担当者一覧を展開 |

### 7-4. フィルター機能

「絞り込み」ボタン（ファネルアイコン）でパネルを開閉します。

| フィルター | 内容 |
|---|---|
| 拠点 | タグに設定された拠点（東京・大阪・福岡など）で絞り込み |
| 状態 | ステータス（★重点顧客など）で絞り込み。複数選択はAND条件 |
| カスタムフィールド | 設定した選択肢型フィールドの値で絞り込み |

> 検索バーでは名前・会社名・メール・役職・電話番号を横断検索できます。

### 7-5. 一括操作

チェックボックスで複数選択した後、画面下部のバーから操作できます。

- **CSVエクスポート**: 選択した顧客のみエクスポート
- **削除**: 選択した顧客を一括削除（元に戻せません）

## 8. 商談・パイプライン管理

### 商談の作成

- 顧客詳細ページ → 「商談」タブ → 「商談を作成」
- 商談一覧ページ → 「新規商談」

### ステージ管理

カンバン形式でステージを移動できます。「受注」「失注」フラグ付きのステージに移動すると分析画面の集計に反映されます。

## 9. トスアップ機能

別事業部への顧客・商談の引き継ぎ機能です。

### 送信

1. サイドバー「トスアップ」ボタン、または顧客詳細「トスアップ実行」
2. 送り先事業部・顧客・メッセージを入力して送信

### 受信側の対応

- 送り先事業部のメンバーのサイドバーに未読バッジが表示される
- **トスアップ一覧** でステータスを管理: 未読 → 対応中 → 完了

## 10. グループアプリ連携

サイドバー下部「グループアプリ」からポロックグループの他アプリへアクセスできます。

アプリの追加・変更は `src/components/layout/Sidebar.tsx` の `GROUP_APPS` 配列を編集してください。

**現在登録済みのアプリ**

| アプリ名 | URL | 説明 |
|---|---|---|
| タレントマネジメント | https://company-management-app-v2.vercel.app | 人材管理・組織 |
| ポロック杯 | https://pollock-cup.vercel.app/cases | 社内大会管理 |

追加する場合は `src/components/layout/Sidebar.tsx` の `GROUP_APPS` 配列に追記してください。

```ts
const GROUP_APPS = [
  {
    name: 'アプリ名',
    url: 'https://example.com',
    emoji: '📊',
    description: '説明文',
  },
]
```

## 11. よくある質問・トラブル対応

### Q. ユーザーが「権限がありません」と言われて顧客を登録できない

**原因**: ユーザーに所属事業部が設定されていません。

**対応**: 設定 → アカウント管理 → 該当ユーザーの ✏️ → 所属事業部にチェックを入れて保存。

---

### Q. ユーザー追加時に「400 Invalid API Key」エラーが出る

**原因**: `SUPABASE_SERVICE_ROLE_KEY` が間違っているかVercelに設定されていません。

**確認手順**:
1. Supabase ダッシュボード → Settings → API → **service_role** のキーをコピー
2. Vercel → Settings → Environment Variables → `SUPABASE_SERVICE_ROLE_KEY` を更新
3. Vercel で再デプロイ

> ⚠️ キーはSupabaseプロジェクトごとに異なります。複数プロジェクトがある場合は正しいプロジェクトのキーを使用してください。

---

### Q. 住所を登録・インポートしても表示されない

**原因**: DBマイグレーションが未実行の可能性があります。

**対応**: Supabase SQL Editor で以下を実行。

```sql
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS address TEXT;
ALTER TABLE contacts ADD COLUMN IF NOT EXISTS department TEXT;
```

---

### Q. CSVインポートで文字化けする

**原因**: ExcelのデフォルトCSV保存はShift-JISです。

**対応**: Googleスプレッドシートで作成してダウンロードするか、Excelで「名前を付けて保存 → CSV UTF-8（BOM付き）」を選択してください。

---

### Q. 顧客一覧のフィルターにカスタムフィールドが出てこない

**確認事項**:
1. 設定 → カスタムフィールドで、フィールドタイプが **「選択肢」** になっているか
2. 選択肢が1つ以上登録されているか

テキスト型・数値型はフィルターに表示されません（選択肢型のみ対応）。

---

### Q. 大量選択で削除が失敗する

最新バージョンでは50件ずつ分割して削除する処理になっています。エラーが続く場合はブラウザをリロードして再試行してください。

---

### Q. インポート中にページを離れてしまった

インポートは途中で止まります。それまでに登録されたデータはDBに残っています。再度同じCSVをインポートする際は **「重複時: スキップ」** を選ぶことで、登録済みのデータを二重登録せずに残りのデータのみ取り込めます。
